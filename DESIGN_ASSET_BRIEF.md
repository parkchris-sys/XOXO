# Send to All — 디자인 애셋 요청서 (전면 개편)

## 아트 디렉션 (필수 공유)
- **컨셉**: 낡은 한국 사내 그룹웨어/전자결재를 게임처럼 과장한 코미디 UI (Agreeee 톤)
- **금지**: 귀여운 모바일 게임 느낌 / 깔끔한 SaaS 대시보드 느낌 / 실제 업무툴처럼 밋밋한 화면
- **지향**: 레트로 그룹웨어 + 게임 HUD, 일부러 촌스럽지만 완성도 높게
- **팔레트**: 다크네이비 `#001B3D`, 스틸블루 `#415A77`, 블루그레이 `#7780A9`, 라이트그레이 `#E0E1DD`, 경고앰버 `#FFD166`, 성공그린 `#06D6A0`, 실패레드 `#EF476F`, 페이퍼 `#F4F1EC`
- **포맷 원칙**: UI 요소는 **SVG**(확대 깨짐 방지), 일러스트/캐릭터/배경은 **PNG(투명)** 권장. 단일 HTML 파일 인라인 삽입이므로 용량 최소화.
- **해상도**: 기준 캔버스 폭 약 360~370px. @2x(720px) 대응 권장.

## 1. 공통 / 글로벌 UI
| 애셋 | 포맷 | 사양 | 비고 |
|------|------|------|------|
| 앱 배경 텍스처 | SVG/PNG | 타일링 가능 | 네이비 + 미세 그리드/노이즈/스캔라인 |
| 게임 로고 (Send to All) | SVG | 가로형, 세로형 2종 | 봉투+뱃지 모티프, 2색(화이트+앰버) |
| 로고 서브 배지 | SVG | 소형 | "전자결재 승인 미션" 등 태그 |
| 공통 패널 프레임 | SVG (9-slice) | 두꺼운 테두리 | 카드/창 공용, 그림자 포함 |
| 스크롤바 | CSS | 얇은 스타일 | 네이비 계열 |

## 2. 화면별 (Screen)
### 2-1. 시작 화면
- 배경 일러스트 (야근하는 직장인/사무실, PNG 투명 또는 풀배경)
- 시작 버튼 (대형 게임 버튼, 앰버)
- 설정 버튼 아이콘
- 미션 안내 클립보드/쪽지 프레임

### 2-2. 게임플레이 HUD (상단바)
- STAGE 카운터 프레임 (예: 05/15)
- 타이머 프레임 (일반/경고 2종 — 경고는 레드 펄스)
- 진행도 바 (빈 바 + 그린 채움 + 광택)
- 결재자(대상자) 뱃지 프레임 + 기본 아바타
- 햄버거/메뉴 아이콘
- 하단 아이템바: 힌트 / 패스 / 시간추가 / 도장 아이콘 (각 4종 + 카운트 뱃지)

### 2-3. 실패 화면
- "반려" 대형 고무도장 (회전, 레드) — SVG
- 화난 상사 캐릭터 일러스트 (PNG)
- 실패 배경 오버레이 (레드 틴트, 찢긴 종이 파편)
- "다시 도전하기" 버튼

### 2-4. 클리어 화면
- "전사 발송 완료" 타이틀 로고
- "승인 완료" 원형 도장 (그린) — SVG
- 환호하는 직장인 캐릭터 일러스트 (PNG)
- 색종이/폭죽 파티클 (PNG 스프라이트 or SVG)
- 결과 요약 패널 프레임
- "처음으로" / "한 번 더" 버튼

### 2-5. 보스전 (Stage 15)
- 보스 캐릭터 일러스트 (대표이사/최종 결재권자, PNG, 표정 2~3종)
- 보스전 전용 배경/프레임
- "전사 발송" 최종 버튼 (특별 연출)

## 3. 문서 / 결재 패널
- 문서 카드 프레임 (종이+전자결재 창 혼합, 두꺼운 테두리)
- 문서 헤더 바 (네이비 그라데이션)
- 상태 뱃지 3종: "동의 대기"(앰버) / "승인"(그린) / "반려"(레드)
- 결재자 정보 행 프레임 + 아바타 세트 (직급별 5~6종)
- 첨부파일 행 프레임 + 파일 아이콘
- "검토 항목" 섹션 구분선/헤더

## 4. 버튼 (상태별 각각)
각 버튼 = 기본 / hover / pressed / 비활성 **4-state**
- 승인(동의합니다) — 그린
- 반려(반대합니다) — 레드
- 확인 — 블루
- 앰버 강조 버튼 (시작/재도전 등)
- 회색 보조 버튼

## 5. 도장 / 배지 (Stamps & Badges) — 전부 SVG
- 승인 / 반려 / 재검토 (원형 도장)
- 완료 도장
- 긴급 / 중요 / 참고 / 완료 (사각 라벨 배지)
- 도장 찍힘 이펙트 (잉크 번짐, 스탬프 프레임)

## 6. 아이콘 세트 (통일된 스타일, SVG)
문서 / 결재함 / 첨부파일 / 사용자 / 경고 / 힌트(전구) / 시간(시계) / 도장 / 메뉴 / 설정(톱니) / 닫기(X) / 체크 / 화살표

## 7. 팝업 / 알림
- 레트로 OS 창 프레임 (네이비 타이틀바 + ■□✕)
- 경고 팝업 (⚠ 아이콘, 타이머 경고용)
- 확인 팝업 (예/아니오)
- 시스템 알림 토스트
- 알림 뱃지 (빨간 원 + 숫자)

## 8. 미니게임별 애셋 (14종 — 전면 개편)

> 공통: 각 미니게임 캔버스 배경 프레임 + 게임 영역 테두리. 조작 영역/충돌 판정 크기는 개발이 유지하니 **비주얼만** 제작.

### 8-1. brick_approval (벽돌깨기)
- 벽돌 스프라이트 (일반/서류더미 모티프, 다단계 2~3종)
- 패들 (결재판/자 모양)
- 공 (도장/서류 클립)
- 배경 프레임

### 8-2. conveyor_approval (컨베이어 벨트)
- 컨베이어 벨트 텍스처 (움직임 대응 타일)
- 서류 박스/버튼 아이템 (정답/오답 구분)
- 수집 존 표시

### 8-3. assemble_approval (문서 조립)
- 문서 조각 퍼즐 피스 세트
- 완성 가이드 실루엣
- 스냅 존 표시

### 8-4. spot_difference (틀린그림찾기)
- 문서 원본/수신본 프레임 2단
- 차이 표시 마커 (동그라미/하이라이트)
- 발견 카운터 (0/3)

### 8-5. whack_approval (두더지 잡기)
- 구멍/책상 서랍 프레임 (6홀)
- 튀어나오는 캐릭터/버튼 (정답 그린 / 오타 함정 / 오답 회색)
- 망치 or 도장 커서

### 8-6. falling_approval (낙하 아이템 받기)
- 낙하 아이템 (서류/버튼, 정답/오답)
- 받는 바구니/결재함
- 배경

### 8-7. catch_approval (박스 기울이기)
- 빈 박스/서랍 프레임
- 회전 화살표 아이콘 (코너)
- 떨어지는 "동의합니다" 버튼 오브젝트
- 기울기 물리 표현용 그림자

### 8-8. tilt_approval (공 굴리기)
- 트레이/미로 배경
- 굴러가는 공 (도장 모티프)
- TARGET 존 (그린) / FAIL 존 좌우 (레드)
- 홀드 게이지 바

### 8-9. escape_approval (장애물 회피)
- 이동 버튼 오브젝트
- 장애물 세트 (벽/집기)
- 목표 지점 표시
- 배경 그리드

### 8-10. excel_approval (엑셀 셀 찾기)
- 스프레드시트 그리드 프레임 (행번호/열문자 헤더)
- 셀 하이라이트/선택 표시
- 엑셀 툴바 장식 (가짜)

### 8-11. parking_approval (주차)
- 회사 건물 (원더피플 간판) 배경
- 주차장 바닥/구획선
- "사장님 자리" 표지판
- 플레이어 차량 (탑뷰) + 주차된 차량 4~5색
- 장애물: 콘, 콘크리트 배리어, 벽
- 홀드/성공 게이지

### 8-12. DecoyRapidButton (페이크 버튼)
- 하단 액션 영역 프레임
- 버튼 상태 배리에이션 (진짜/페이크 시각 구분은 최소화 — 개발 조율)
- 숨은 버튼 배치용 보조 UI (배지/브레드크럼/첨부헤더 등)

### 8-13. forbidden_zones (Flappy Bird형)
- 날아가는 캐릭터 (서류봉투/직장인)
- 파이프/장애물 (서류 캐비닛 모티프)
- 스크롤 배경 (사무실 복도)
- 목표 지점 (골)

### 8-14. running_button (도망가는 버튼)
- 도망치는 "동의합니다" 버튼 (표정/다리 등 캐릭터화)
- 커서 추적 표현

## 우선순위 (제작 순서 제안)
1. **P0 — 공통**: 팔레트, 로고, 패널 프레임, 버튼 4-state, 도장 세트, 아이콘 세트, HUD
2. **P1 — 화면**: 시작/실패/클리어 화면 일러스트 + 프레임
3. **P2 — 미니게임**: 사용 빈도 높은 순 (parking, whack, tilt, brick, catch, excel...)
4. **P3 — 보스전 + 이펙트/사운드**

## 전달 시 명시할 것
- 파일명 규칙 (예: `mg_parking_car_red.svg`)
- 애셋 매니페스트 (용도/위치 매핑) 함께 요청
- 다크 배경 위 올라가는 요소는 투명 배경 필수
